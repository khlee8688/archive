This post documents the process of integrating **Unity** with **ROS Melodic** using **Docker and WSL2**, specifically for controlling the **OMO R1 mobile robot**. The goal is to set up an environment where Unity can serve as a visualization or control interface, sending commands to and receiving data from the robot via ROS. This setup is useful for simulating robot behavior, testing control logic, or extending physical robot interfaces with interactive 3D environments.

We’ll walk through the recommended installation order, focusing on the **Docker-based approach on Ubuntu 24.04 via WSL2**, which allows us to use ROS Melodic even on unsupported modern systems.

---  
## Installation Order  
  
**Method 1: VirtualBox → Ubuntu → Docker → ROS Melodic**  
  
**Method 2: WSL 2 → Ubuntu (18.04 or 24.04) → Docker → ROS Melodic (this one)**  
  
> ROS Melodic can only be officially installed in Ubuntu 18.04 (Bionic).  
> Docker installation method is used in the latest Ubuntu (20.04, 22.04, 24.04).
