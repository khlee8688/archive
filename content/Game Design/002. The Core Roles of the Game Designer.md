In game development, the designer takes on the role of setting the direction of the entire team. Beyond simply coming up with ideas, it organizes the structure, defines the rules, and helps development and design move under one goal. In this article, the core roles that the planner should play in the actual development process are summarized step by step.  
  
---  
  
## 1. Organize the structure of the game and share it with the team  
  
The first thing a planner needs to do is to get the full outline of the game and share it with team members. If developers do not understand 'what to make and how to make it', confusion will inevitably arise during the development process.  
  
**What the designer needs to clearly organize:**:  
- **The goal of the game**  
	- What kind of experience does this game want to offer  
	- Example: Multi-play Survival Game to Maximize Fear and Tension  
- **Core Systems**  
	- Key features that must be implemented  
	- e.g. Map navigation, Monster AI, Player interaction, etc  
- **Basic rules and mechanisms of the game**  
	- Example: Monsters don't chase players for more than a certain distance  
- **UX Flow (Order of Player Experience)**  
	- Example: Main Menu → Robbie → Game Start → Survive or Escape  
  
If all these contents are organized into a single document or slide and shared with the entire team, developers will not be confused about "what to make."  
  
---  
  
## 2. Create documents that developers can understand  
  
The planner should go beyond simply determining the direction and create practical documents** for developers to refer to. It is not a vague explanation, but it should contain standards and structures that can be implemented.  
  
**Key document examples:**:  
- Basic Functional Plan  
- UI Plan (including specification for front-end developers)  
  
By writing the document neatly and clearly, developers can quickly understand "why make this and what action to do."  
  
---  
  
## 3. Setting development priorities and managing schedules  
  
When multiple developers work at the same time, **What to create first and what functions to implement later** must be clearly determined. Otherwise, a conflict will occur as the team members develop different functions.  
  
**Priorities that need to be sorted out by the planner:**:  
- **MVP Definition (Minimum Functional Product)**  
	- Examples: Multi-play capability, Monster AI, lighting system  
- **Dividing the development phase**  
	- Step 1: Basic Character Movement & Map Implementation  
	- Step 2: Implement Monster AI  
	- Step 3: Add puzzles and interaction systems  
	- Step 4: Balancing  
- **Using Kanban boards (Trello, Notion, Jira, etc.)**  
	- Real-time management of personnel and status for each function  
  
If no priorities are set, developers may start with the wrong functions and have a twisted schedule. The designer is in charge of designing and managing this flow.  
  
---  
  
## 4. Continuous feedback and communication  
  
Planning does not end with creating documents. The ability to continuously **explain planning intentions, correct directions, and respond in real time is important during the development process.  
  
**What the designer should do in communication:**:  
	- Ensure features are being implemented as intended  
	- Share with teams quickly if there is a plan change  
	- Periodic team meetings to monitor progress  
  
In particular, as soon as developers ask, "How do I do this?" during functional implementation, they should be able to answer immediately. The planner should serve as a communication hub for the whole team, not just a designer.  
  
---  
  
## Tidying up: Four things an organizer must do  
  
1. Clear and share the entire structure of the game  
2. Create a planning document for developers to refer to  
3. Set development priorities and coordinate schedules  
4. Continuous communication and problem solving with developers  
  
If designer do not provide clear guidelines, developers may work in different directions, resulting in **unnecessary functions** or **non-functional systems**. The designer plays a key role in preventing this confusion and helping the team move efficiently toward a single goal.
